How this blog is built

When I started to think of writing a Front End related blog, the first thoughts that came to my mind were about the tech stack I would need.

I established three clear conditions for the stack to cover:

  • To run on JavaScript. Makes sense for a Front End related blog.
  • To be as simple and as “client-only” as possible. No databases, bloat, server-side code and so on.
  • To run in a free environment. I don’t really want to get a VPS or anything to host this.

At first though, and because of my personal experiece, the most obvious answer was WordPress. I made some WP sites (blogs and otherwise) back in the day and I was a pretty advanced user. However, this choice didn’t really cover the conditions, so I had to move on.

Right after WP, I layed my eyes on Ghost. I liked the choice at first because it runs on Node.js and all the cool kids are using it and it has a lot of nice themes and stuff ready to use and all…

It turned out Ghost didn’t comply with my second and third conditions because you need an Amazon EC instance or a VPS or something you can install Ubuntu and Node.js and Nginx and maybe something more, which makes it not simple enough (and probably not free either!)

I explored some other options out there like Jekyll (free hosting through GH Pages, but no JS!), Poet (which incorporates cool stuff like routing but needs a Node.js backend and has an ugly as hell homepage!) and more. Then, finally, I came across Hexo.

Why Hexo?

Basically, because it covers my three conditions.

  • Runs completely on JS.
  • Posts are saved as Markdown files. No database, no server, no nothing; just good ol’ client-side code. Local development is based on Node.js with a task runner a-la-Grunt, but it’s not even necessary to run the server command if you’re just writing a post.
  • Deployment to GitHub Pages is easy as running a command in the console. And yes, GitHub Pages is a free and pretty cool client-side hosting.

What else does this site use?

This blog’s theme is a slightly modified version of the Hexo-ported Casper theme. It also uses jQuery, of course; the header backgrounds are randomly made on the fly through the very cool Trianglify library by Quinn Rohlf.

The typography is Open Sans for the titles and Noto Serif for the body. Writting and development is powered by caffeine and Spotify.

That’s pretty much it, I think. I will surely make frequent changes to the blog, and I’ll try to comment them all. Your comments and questions are welcomed.
